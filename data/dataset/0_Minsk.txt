Smartdash нь хэрэглэгчийн сэтгэл ханамжийг тухай бүрд бодитоор үнэлж дүгнэх боломжтой. Мөн сошиал сурталчилгааны үр дүнгээ шууд хэмжин, хэрэглэгчийн сэтгэл ханамжийг нэмэгдүүлэх төлөвлөгөө боловсруулахад энэхүү сошиал аналитик үйлчилгээ юм.
    1. Сэтгэгдлийн Шинжилгээ

Зорилго: Фэйсбүүк дэх постуудын сэтгэгдлийг автоматаар шинжлэх, ангилах.
Функционал: Сэтгэгдлийг эерэг, сөрөг, тэнцвэртэй гэх мэтчилэн ангилахын тулд орчин үеийн
хэлний боловсруулалтын алгоритмуудыг ашиглах.
Мэдэгдэл: Сэтгэгдлүүдийн төлөвлөгөөг товч тайлбарлах тайлан гаргах.
2. Сөрөг Санал Хүсэлтийг Тайлагнах
Зорилго: Сөрөг санал хүсэлтийг шуурхай мэдээлэх.
Функционал: Сөрөг сэтгэгдлийг илрүүлж, шуурхай хяналтанд авах.
Мэдэгдэл: Сөрөг санал хүсэлтүүд тухайн төслийн холбогдох хүмүүст шуурхай мэдэгдэх систем
бүрдүүлэх.
3. Олон Хуудсыг Удирдах
Зорилго: Олон Фэйсбүүк хуудсыг нэгтгэн удирдах.
Функционал: Олон Фэйсбүүк хуудасны сэтгэгдэл, постуудыг интеграцчлах, зэрэгцүүлэн шинжлэх.
4. Хэрэглэгчийн Хандлагын Шинжилгээ
Зорилго: Брэнд, бүтээгдэхүүн эсвэл сурталчилгааны талаар хэрэглэгчийн ерөнхий хандлагыг ойлгох.
Арга Зүй: Хэрэглэгчийн хандлагыг тодорхойлохын тулд сэтгэгдлийн анализийн өгөгдлийг нэгтгэх.
Тайлан: Хугацааны туршид хэрэглэгчийн хандлагын чиг хандлагыг харуулах бүрэн тайлан гаргах. 5. Маркетингийн Аяны Шинжилгээ
Зорилго: Фэйсбүүк дэх маркетингийн аянуудын нөлөөллийг, хүлээн авалтыг хэмжих Бусад функцууд:

3.3 DATABASE STRUCTURE
TO WHOM: ITS, TOMS
/ Оруулах хэлбэр: Өгөгдлийн сангийн хүснэгтийг логик бүтцийг тодорхойлж харуулах. Дэлгэрэнгүй хэсэг, script зэргийг тусдаа хуудас үүсгэнэ гэж тооцох. /
  
 1. Comments Table:
Зорилго: Хэрэглэгчдийн нийтлэл дээр бичсэн сэтгэгдэл шууд доорх бүтцийн дагуу хадгална.
Бүтэц:
id : Unique identifier for each comment.
comment : Identifier linking to the associated post.
post_id : Identifier linking to the user who made the comment. created_time : The actual text of the comment.
2. Predicted Comments Table:
Зорилго: Хэрэглэгчдийн нийтлэл дээр бичсэн сэтгэгдэл шууд доорх бүтцийн дагуу хадгална. Бүтэц:
page_id : Unique Facebook page identifier. id : Unique comment identifier.
comment : Text of the Facebook comment. label : Classification tag for the comment.

post_id : Identifier of the related Facebook post. category : Comment category or type.
post_ct : Timestamp of post creation. confidence : Confidence level of the label. created_time : Timestamp of comment creation. expire time : Timestamp for data expiration.
3. Social_account Table:
Зорилго: Хэрэглэгчдийн сошиал хаяг доорх бүтцийн дагуу хадгална.
Бүтэц:
name : The name associated with the social account.
account_id : Unique identifier for the social account.
company : The company or organization linked to the account.
account_type : Type or category of the social account (e.g., personal, business, celebrity). created_time : Timestamp when the social account was created.
expire_time : Timestamp indicating when the account data may expire or become inactive. updated_time : Timestamp indicating the last update time for the social account information.
4. Posts Table:
Зорилго: Хэрэглэгчдийн нийтэлсэн доорх бүтцийн дагуу хадгална.
Бүтэц:
id : Unique identifier for the post.
message : The content or message of the post.
category : The category or type associated with the post.
created_time : Timestamp when the post was created.
expire_time : Timestamp indicating when the post may expire or become inactive. updated_time : Timestamp indicating the last update time for the post.
5. Posts category Table:
Зорилго: Хэрэглэгчдийн нийтлэл дээр бичсэн сэтгэгдэл шууд доорх бүтцийн дагуу хадгална.
Бүтэц:
id : Unique identifier for each comment.
comment : Identifier linking to the associated post.
post_id : Identifier linking to the user who made the comment. created_time : The actual text of the comment.
6. Invitation Table:
Зорилго: Хэрэглэгчдийн урисан мэдээлэл доорх бүтцийн дагуу хадгална.
Бүтэц:
id : Unique identifier for each comment.

comment : Identifier linking to the associated post.
post_id : Identifier linking to the user who made the comment. created_time : The actual text of the comment.
7. Customer Table:
Зорилго: Хэрэглэгчдийн мэдээллийн дээр бичсэн шууд доорх бүтцийн дагуу хадгална.
Бүтэц:
id : Unique identifier for the customer.
pages : Number of pages associated with the customer. company_id : Customer’s associated company ID firstname : First name of the customer.
last : Last name of the customer.
password : Password associated with the customer's account.
email : Email address of the customer.
phone : Phone number of the customer.
id_authority : Identifier for the customer's authority or permissions.
created_time : Timestamp when the customer's account was created.
updated_time : Timestamp indicating the last update time for the customer's information.
8. Payment Table:
Зорилго: Төлбөр тооцооны доорх бүтцийн дагуу хадгална.
Бүтэц:
id : Unique identifier for the payment transaction.
account_id : Identifier of the associated customer or account.
amount : The payment amount.
extend_duration : Duration or period extended due to the payment. vat_lottery_no : VAT (Value Added Tax) lottery number related to the payment. vat_qr : QR code associated with VAT information.
vat_bill_no : VAT bill number for the payment.
9. Companies Table:
Зорилго: Хэрэглэгч байгууллагуудын пэйж хуудас, мэдээллийг хадгална.
Бүтэц:
id : Unique identifier for the payment transaction.
label : Facebook page title, label
pages : Facebook pages information list (List<{ page_id: string, label: string }>
